Kuka soittaa? (Jul 2023 – )

I made a jokeful mobile app where a random beverage calls the user at a random time, every day. The user has the option to either answer the call or decline it. The app shows how long it took for each user to answer or decline the call and a comment from the user. Users can give likes to others' comments and the app shows how many likes each comment has. I also made an admin panel for managing what beverage is calling and when. New beverages can be dynamically added to the app through the admin panel. The project was inspired by the popular Be Real app.

  • Backend implemented using Node, ExpressJS and Prisma with a PostgreSQL database
  • Frontend implemented using Typescript and React Native with Expo
  • Admin panel implemented using React
  • The app was deployed to Hetnzer Cloud using Docker

Project repository: https://github.com/joonatan-aatos/kuka-soittaa

Jalmarin Eriskummalliset Seikkailut (JES) (Feb 2021 – Mar 2022)

I made a fast-paced multiplayer platforming game with a couple of my friends. We have named it Jalmarin Eriskummalliset Seikkailut, meaning Jalmari's bizarre adventures. Our goal was to create a multiplayer video game that can be played on any modern web browser.

  • Back-end implemented in C++ using the websocket++ and nlohmann/json libraries
  • Front-end implemented in JavaScript
  • Multithreaded socket programming
  • Contains server process and multiple clients running on different computers

Backend repository: https://gitlab.paivola.fi/jkorpela/jes-back-end

Frontend repository: https://gitlab.paivola.fi/jkorpela/jes

Car simulation (TuKoKe project) (Sep 2021 – Jan 2022)

I and my firend, Justus Hansen, participated in the annual science competition for high school students, TuKoKe, with a study about the sufficiency of the Finnish electric car charger infrastructure. We made a model of the Finnish E75-road where we simulated finnish electric cars.

  • Implemented in Java
  • Simulations were run simultainiusly in different threads
  • Different algorithms were implemented to optimize the behaviour of cars
  • We made a web scraper to collect different statistics of car models with Celenium
  • We made a separate program to parse our programs' output

Car simulation repository: https://gitlab.paivola.fi/jkorpela/saehkoeautosimulaatio

Web scraper repository: https://gitlab.paivola.fi/jkorpela/car-parser

Output parser repository: https://gitlab.paivola.fi/jkorpela/car-simulation-output-parser
